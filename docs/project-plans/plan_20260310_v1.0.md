# ASM 구축 프로젝트 상세 계획서 (v1.0)

## 1. 개요
본 문서는 외부(External) 및 내부(Internal) 공격 표면 관리(Attack Surface Management, ASM) 시스템을 구축하기 위한 상세 실행 계획을 담고 있습니다. 기존 계획을 바탕으로 **자동화 효율성**, **데이터 가시성**, **보안성**을 강화한 개선안을 제안합니다.

## 2. 주요 검토 및 개선 사항 (Technical Review)

### 2.1 외부(External) ASM 고도화
*   **자산 탐지 속도 및 정확도**: 
    *   `Amass`는 강력하지만 속도가 느리고 리소스를 많이 소모합니다. `assetfinder`를 병행하여 상보적으로 사용하고, 대량 도메인 처리를 위해 `massdns` 도입을 권장합니다.
    *   **개선안**: `Subfinder` -> `Assetfinder` -> `Massdns` -> `Dnsx` 파이프라인 구성.
*   **Web Visual Recon**: 단순히 텍스트 스택만 확인하는 것이 아니라, `gowitness` 또는 `playwright` 기반 스크린샷 캡처 기능을 추가하여 대규모 자산을 시각적으로 빠르게 검토할 수 있도록 합니다.
*   **Nuclei 효율화**: 모든 템플릿을 돌리는 것은 비효율적입니다. `httpx`에서 탐지된 기술 스택(Technology)을 기반으로 Nuclei의 태그(`-tags`) 필터링을 사용하여 맞춤형 스캔을 수행합니다.

### 2.2 내부(Internal) ASM 보안 및 범위
*   **스캐너 보안**: 내부 네트워크에 상주하는 스캐너는 공격자에게 가장 매력적인 타겟입니다. 스캐너 자체의 보안(SSH 키 관리, 로그 격리)을 최우선으로 고려합니다.
*   **네트워크 프로토콜 확장**: `Nmap` 외에도 `Responder` 모드(패시브 모니터링)를 활용하여 LLMNR/NBT-NS 중독 공격 시도를 탐지하는 기능을 검토합니다.

### 2.3 데이터 모델 및 플랫폼 (Integration)
*   **Schema 유연성**: 자산 데이터는 기술 변화에 따라 속성이 다양해집니다. PostgreSQL의 `JSONB` 타입을 적극 활용하여 유연한 데이터 저장을 지원합니다.
*   **Timeline 관리**: 자산의 '변경 사항'을 포인트 단위가 아닌 타임라인으로 관리하여, 특정 시점에 새로 열린 포트나 추가된 서브도메인을 추적할 수 있도록 설계합니다.

## 3. 단계별 구축 가이드

### 1단계: External ASM (Infra Setup)
*   **Tools**: Subfinder, Dnsx, Naabu, Httpx, Nuclei
*   **Environment**: 각 도구는 Docker 컨테이너화하여 의존성 충돌 방지
*   **Automation**: Python `Celery` 또는 간단한 `Task Queue`를 사용하여 비동기 스캔 처리

### 2단계: Internal ASM (Deep Scan)
*   **Tools**: Masscan (빠른 탐지), Nmap (정밀 스캔)
*   **Security**: 스캔 대상 인프라에 부하를 주지 않도록 Rate-limit 설정 준수

### 3단계: Integration (Platform)
*   **Backend**: FastAPI (스캔 명령 전달 및 결과 조회 API)
*   **DB**: PostgreSQL (자산 인벤토리 마스터)
*   **Alert**: Slack Webhook 통합

## 4. 권장 기술 스택 및 구조
*   **언어**: Python 3.12+ (uv 패키지 매니저 권장)
*   **오케스트레이션**: Docker Compose
*   **정적 코드 분석**: Ruff (Lint/Format)
*   **데이터베이스**: PostgreSQL

## 5. 실행 환경 구축 가이드 (Windows 11 Hyper-V 기반)
본 프로젝트는 네트워크 스캐닝의 성능 및 정확도를 극대화하고, 호스트 OS를 보호하기 위해 **독립된 Linux 가상 환경**에서의 실행을 권장합니다. Windows 11 사용자의 경우 내장된 Hyper-V를 활용하여 손쉽게 최적의 환경을 구축할 수 있습니다.

### 5.1 Hyper-V Ubuntu 가상 머신 생성
1. Windows 검색창에 `Hyper-V 빨리 생성(Hyper-V Quick Create)`을 검색하여 관리자 권한으로 실행합니다.
2. 운영체제 목록에서 **Ubuntu 22.04 LTS** (또는 최신 LTS 버전)를 선택하고 `가상 컴퓨터 만들기`를 클릭합니다.
3. 생성 완료 후 `연결` 및 `시작`을 클릭하여 초기 계정(Ubuntu 사용자명/비밀번호)을 설정합니다.

### 5.2 최적화 설정 (네트워크 및 리소스)
* **네트워크 스위치 변경**: Hyper-V 가상 스위치 관리자에서 `외부(External) 가상 스위치`를 생성하고 VM에 할당합니다. (스캔 시 NAT로 인한 방화벽 차단 방지 및 정확한 IP 매핑 목적)
* **리소스 할당**: `Nuclei` 등 병렬 스캐너의 부하를 견디기 위해 최소 **4 Core vCPU**와 **8GB 이상의 RAM** 할당을 권장합니다.

### 5.3 프로젝트 실행 방법
1. 생성된 Ubuntu 터미널에 접속하여 본 저장소를 클론(Clone)합니다.
2. `Docker` 및 `Docker Compose`를 설치합니다. (또는 `uv`를 통한 Python 환경 세팅)
3. 프로젝트 루트 디렉토리에서 환경 변수(`.env`)를 설정한 후, `docker-compose up -d` 명령어를 통해 통합 ASM 플랫폼을 컨테이너 환경에서 안전하게 실행합니다.

## 6. 보안 가이드라인
*   API Key(Shodan, GitHub 등)는 반드시 `.env` 파일로 격리하고 `Varlock` 등 안전한 방식을 통해 관리합니다.
*   스캔 타겟 환경 및 범위를 명확히 설정(`targets.yaml`)하고, 인가받지 않은 외부 자산에 대한 악의적인 스캔을 엄격히 금지합니다.
